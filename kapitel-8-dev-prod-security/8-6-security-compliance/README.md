# Case Study 8.6 – Docker Security and Compliance

This example summarizes Docker security and compliance measures:

- trusted base images
- multi-stage builds
- non-root container execution
- minimized Linux capabilities
- read-only filesystem
- temporary writable directories via tmpfs
- resource limits
- vulnerability scanning with tools such as Trivy
- centralized logging and audit documentation

The goal is to reduce the attack surface and support traceability, compliance and operational security in containerized environments.
